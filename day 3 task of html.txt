1. What Are Semantic Tags?

Ans: Semantic HTML tags clearly describe the meaning of the content inside them.
Examples of semantic tags:
	<header>
	<footer>
	<article>
	<section>
	<nav>
	<main>
Example:

	<header>
  		<h1>My Website</h1>
	</header>

Here the browser, search engines, and screen readers know that this section is the header of the page.

2. What Are Non-Semantic Tags?

Ans: Non-semantic tags do not describe the content meaning. They only help with layout or grouping.

Examples:

	<div>
	<span>

Example:

	<div>
  	   <h1>My Website</h1>
	</div>

Here we don't know if the <div> is a header, footer, sidebar, or something else.

3. Semantic vs Non-Semantic Tags:

| Feature       | Semantic Tags                       | Non-Semantic Tags          |
| ------------- | ----------------------------------- | -------------------------- |
| Meaning       | Describe the content                | No meaning                 |
| SEO           | Better for search engines           | Not helpful                |
| Accessibility | Screen readers understand structure | Harder for assistive tools |
| Examples      | `<header>`, `<footer>`, `<article>` | `<div>`, `<span>`          |

4. Why Semantic Tags Are Important:
1)Better SEO

Ans: Search engines understand your page structure better.

Example:

	<header> → site title
	<nav> → navigation menu
	<article> → main content
	<footer> → contact info

2) Better Accessibility

Ans: Screen readers used by visually impaired users rely on semantic structure.

3)Cleaner Code

Ans: Developers can easily understand the layout.
